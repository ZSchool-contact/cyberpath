הסיפורים מאחורי המספרים

לחצו על כל כרטיס לסיפור שיגרום לכם לזכור

22 ← 23
SSH "דחף" את Telnet
למה SSH קיבל את המספר 22 בדיוק?
ב-1969 Telnet קיבל את פורט 23. הוא שלט בגישה מרחוק במשך 26 שנה.

ב-1995 Tatu Ylönen המציא SSH כדי להחליף את Telnet — כי Telnet מעביר הכל בטקסט גלוי, כולל סיסמאות.

הוא בחר פורט 22 במכוון: ממש לפני 23. "SSH באה לדחוף את Telnet החוצה."
וו זיכרון: 22 ← 23. החדש יושב ממש לפני הישן.
8080
8080 = 80 + 80
למה לא פשוט 81 או 82?
כשפורט 80 תפוס (למשל, שרת HTTP כבר רץ שם), מפתחים צריכים פורט חלופי לשרת נוסף או ממשק ניהול.

8080 נבחר כי זה ממש "80 כפול" — קל לזכור שזה גרסה שנייה של 80. מפתחים ראו את זה כבדיחה פנימית שהפכה לקונבנציה.
וו זיכרון: 8080 = הכפלת 80. אח קטן של הכניסה הראשית.
443
HTTP + מנעול = 443
למה לא 81 לאתר מאובטח?
כש-HTTPS נוצר ב-1994, 80 כבר שייך ל-HTTP. צריך פורט חדש לחלוטין — לא 81 (שנראה כמו שגיאה של 80).

443 נבחר כמספר זמין שרחוק מספיק מ-80 כדי להיות ברור שזה שירות שונה לגמרי. ה-S ב-HTTPS = Secure.
וו זיכרון: 443 = 80 עם שכבת אבטחה. יש לו S — יש לו מנעול.
3306
MySQL בחר "אקראי"
ולמה זה מסוכן כשפתוח?
מפתחי MySQL בחרו 3306 — אין סיפור מיוחד מאחורי המספר. אבל יש חשיבות ענקית:

מסד נתונים אף פעם לא אמור להיות נגיש מחוץ לשרת עצמו. רק האפליקציה שרצה על אותו מחשב צריכה גישה אליו.

אם אתם רואים 3306 פתוח בסריקה — זה אחד הממצאים החמורים ביותר.
וו זיכרון: 3306 = הכספת במרתף. אם פתוחה לרחוב — קריאת חירום.

ההבדל הקריטי

SERVICE vs VERSION

SERVICE
מה nmap מצפה לפי מספר הפורט.

nmap מסתכל במסד הנתונים שלו: "פורט 22? כנראה SSH".

זה ניחוש מבוסס — לא אמת מוחלטת.
VERSION
מה באמת רץ שם.

nmap שולח בקשה לשירות ושואל: "מי אתה?" השירות עונה עם הגרסה.

דורש פקודה: nmap -sV

למה זה חשוב?

בלי -sV:
80/tcp open http
כל מה שיודעים: יש שרת אינטרנט. איזה? לא ידוע.
עם -sV:
80/tcp open http Apache httpd 2.4.7
עכשיו יודעים: Apache גרסה 2.4.7 — גרסה ישנה מ-2013 עם חולשות ידועות. זה ממצא אמיתי.
🎯 מגן מחפש גרסאות ישנות כדי לעדכן. תוקף מחפש גרסאות ישנות כדי לנצל. אותה פקודה — כוונה שונה.

סיכום

מה למדנו ביחידה 6

הבניין הדיגיטלי: כל פורט = דלת. 22 = דלת אחורית מאובטחת. 23 = דלת ישנה בלי מנעול. 3306 = כספת שאסור לרחוב. 8080 = 80+80.
22 לפני 23: SSH נוצר ב-1995 להחליף את Telnet — המפתח בחר 22 כי זה ממש לפני 23. סיפור אמיתי שעוזר לזכור.
SERVICE ≠ VERSION: SERVICE = ניחוש לפי מספר פורט ידוע. VERSION = מה באמת רץ — דורש -sV ומכיל את המידע הקריטי לאבטחה.
דגלים: -sn (no ports), -sV (Version), -sS (SYN), -p (ports), -Pn (skip Ping). כל אות היא קיצור של המילה.
latency: 0.001s = רשת מקומית. 0.15s = שרת בחו"ל. latency גבוה = מרחק פיזי.
